это Android-приложение для распознавания продуктов на фото с помощью ML Kit и последующего поиска рецептов, содержащих распознанный ингредиент.\
Приложение позволяет сделать снимок на камеру или выбрать изображение из галереи, после чего ML Kit определяет объекты на фото, а пользователь может подтвердить или 
уточнить название продукта и найти подходящие рецепты из встроенной базы данных SQLite.

Инструкция по сборке: для сборки проекта необходимо клонировать репозиторий перейти в папку проекта. Открыть проект в Android Studio 
После открытия синхранизовать с Gradle.Готовый APK-файл будет находиться по пути app/build/outputs/apk/debug/app-debug.apk. 
Установить приложение на устройство 

Окружение: для разработки и сборки требуется Android Studio, JDK версии 11 или выше, Gradle 8.2, Android SDK с API 36, NDK версии 17. 
Основные зависимости: androidx.core.ktx, androidx.appcompat, com.google.android.material, com.google.mlkit:text-recognition,
com.google.mlkit:image-labeling версии 17.0.7 и com.google.mlkit:image-labeling-custom версии 17.0.1. 
Устройство для запуска должно работать на Android 7.0 или выше и иметь камеру.

Конфигурация: приложение запрашивает разрешение android.permission.CAMERA для работы с камерой, разрешение запрашивается динамически при первом использовании.
Для выбора изображений из галереи дополнительные разрешения не требуются.
Приложение не использует внешние API-ключи - все компоненты ML Kit и база данных работают локально.
ML Kit автоматически загружает необходимые модели при первом запуске, поэтому для корректной работы распознавания требуется 
интернет-соединение при первом использовании.
База данных SQLite (recipes.db) создаётся автоматически при первом запуске приложения и заполняется
пятью предустановленными рецептами: омлет, яичница, жареная картошка, овощной салат, блины. При необходимости базу можно расширить, 
добавив рецепты в метод insertSampleRecipes класса DatabaseHelper. Никаких дополнительных файлов конфигурации, переменных окружения или правки конфигов не требуется.
